# 小红书自动生成入口设计

## 目标

在当前主首页 `/` 的正式版机器人列表中新增“小红书自动生成”入口。用户点击后，在新标签页打开 `https://xhstw.qycm.top/`。

## 范围

- 只修改主首页 `frontend/app/page.tsx` 及对应的针对性测试。
- 入口名称为“小红书自动生成”，归入“小红书”分类。
- 入口沿用现有正式版机器人卡片的样式与搜索行为。
- 入口要求用户先登录，与现有正式版外部工具保持一致。
- 不修改仍处于草稿状态的 `/home2` 页面。

## 交互与数据流

1. 首页从机器人入口数据中渲染“小红书自动生成”卡片。
2. 未登录用户点击时，跳转登录页，并保留返回入口所需的目标信息。
3. 已登录用户点击时，通过浏览器新标签页打开 `https://xhstw.qycm.top/`。
4. 外部站点打开失败时，主站保持不变；浏览器负责显示外链打开结果。

## 实现方式

为首页机器人元数据增加一个明确的外链字段，并在现有 `openBot` 分支中处理该字段。这样不会把绝对 URL 交给 Next.js 内部路由器，也不会为一个简单外链引入中转页面或 SSO 流程。

## 验证

- 新增测试，先证明当前代码缺少该入口及外链处理，再实现使其通过。
- 验证入口名称、分类、登录要求和目标 URL。
- 运行相关前端测试、ESLint 和 TypeScript/构建校验。

## 非目标

- 不改造目标网站。
- 不新增 SSO、令牌交换或环境变量。
- 不调整其他机器人、首页布局或视觉样式。
- 不部署或推送到远程仓库。
