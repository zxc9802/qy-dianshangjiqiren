# 站内智能体聊天预览版设计

日期：2026-07-20

状态：设计已确认，等待书面审阅

设计方向：绿意专注工作台

## 1. 背景与目标

新版首页已经形成浅绿色、轻量、专注的视觉语言，但走站内 `/chat/[id]` 的非 SSO 智能体仍使用原聊天界面，体验不一致。现有聊天页同时承担大量真实能力，包括流式对话、历史记录、收藏、报告、模型与联网模式、附件、语音、绘图和视频复用，因此本次不能只做静态换皮。

本次先新增同技术栈、连接真实功能的独立预览路由 `/chat2/[id]`，供体验确认。当前 `/chat/[id]` 不修改；用户确认新版后，再单独设计正式替换与代码收敛方案。

## 2. 范围

### 2.1 包含

- 所有使用站内聊天路由的内置和自定义智能体。
- 重点验收智能体：
  - `/chat2/35`：起芽成长特助；
  - `/chat2/37`：视频拆解导演；
  - `/chat2/36`：通用聊天入口。
- 真实登录、会话、收藏、报告、上传、语音、模型、联网、绘图和视频历史能力。
- 桌面端与移动端响应式设计。
- 浅绿色主题及与新版首页一致的视觉语言。

### 2.2 不包含

- 不修改或重定向当前 `/chat/[id]`。
- 不修改 SSO 外链智能体的内部页面。
- 不新增后端接口、数据库表或鉴权机制。
- 不恢复首页工作流入口。
- 不在本阶段推送 GitHub 或替换生产聊天页。

## 3. 技术与隔离策略

- 使用现有 Next.js 16 App Router、React 19、TypeScript、CSS Modules、Zustand 和 `next-themes`。
- 新增独立动态路由 `frontend/app/chat2/[id]/`。
- 复用现有 stores、API client、聊天模型配置、附件解析、流式事件、富文本格式化、本地视频缓存和语音录制模块。
- 预览阶段不重构现有 3127 行聊天页，不改变其运行行为。
- 新页面可以复用或迁移已验证的交互处理逻辑，但不得复制后端协议或创建第二套数据格式。
- 若预览获批，正式替换阶段应将共同的业务控制逻辑抽离为共享模块，避免长期维护两套聊天实现。

## 4. 页面结构

### 4.1 整体布局

- 全屏浅绿色纸张感背景，使用克制的纹理、柔和边线与阴影。
- 聊天主体最大宽度约 920–980px，保证长回答、表格和报告可读性。
- 顶部工作栏固定，消息区独立滚动，底部输入台固定。
- 历史记录默认不占用页面宽度，通过左侧抽屉按需出现。
- 能力设置默认隐藏，通过右侧抽屉按需出现。

### 4.2 顶部工作栏

左侧包含：

- 返回新版首页；
- 当前智能体图标、名称和在线状态；
- 点击智能体名称打开站内智能体切换列表。

右侧包含：

- 新对话；
- 生成报告；
- 历史记录；
- 管理员可见的智能体管理入口。

### 4.3 空白会话

- 显示当前智能体的名称、说明和欢迎语。
- 显示 3 个与当前智能体相关的起始问题。
- 起始问题点击后直接进入真实发送流程。
- 用户开始对话后，欢迎区收起，释放消息空间。

### 4.4 消息区

- 用户消息使用右侧紧凑气泡，不占满内容宽度。
- 助手回答使用左侧宽版文档卡片，不使用传统窄气泡。
- 文档卡片支持标题、段落、列表、表格、代码、图片、附件、视频关键帧和联网来源。
- 长会话继续使用现有分段渲染机制，并提供“显示更早消息”。
- 流式输出保持稳定；用户主动上滑后暂停自动跟随。

### 4.5 输入台

默认只显示：

- 上传按钮；
- 语音按钮；
- 可自动增高的文本输入框；
- 发送按钮；
- 简短能力状态摘要；
- “能力设置”入口。

输入台不得横跨整屏形成冗长工具条，其宽度跟随聊天主体。

## 5. 核心交互

### 5.1 能力设置

桌面端从右侧滑出，移动端使用底部弹层。内容包括：

- 回答模型选择；
- 联网搜索模式；
- 绘图模式；
- 视频拆解智能体专属的历史视频选择。

输入台只显示类似“GPT-5.4 · 联网自动”的当前状态摘要。设置变化沿用现有本地存储键，关闭面板后继续聊天。

### 5.2 历史记录

历史记录从左侧抽屉打开，包含：

- 当前智能体的聊天记录；
- 收藏记录；
- 报告标记与打开入口；
- 收藏、删除等现有操作；
- 新建对话入口。

关闭抽屉后主聊天区恢复完整宽度。移动端抽屉覆盖内容，并提供明确关闭操作。

### 5.3 附件与媒体

- 保留文件选择、拖拽上传和剪贴板粘贴。
- 支持现有图片、视频、PDF、DOCX、TXT、Markdown、CSV 和 PPTX 类型。
- 一次最多 10 个附件。
- 发送前显示紧凑预览、文件名和移除操作。
- 视频拆解智能体保留历史视频复用、最多选择 2 个视频和本机缺失提示。

### 5.4 消息与报告

- 保留真实流式文本、图片结果、来源链接和附件展示。
- 保留表格复制能力。
- 保留生成报告入口及原报告数据流程。
- 工作流不作为新界面的可见入口；旧链接带有工作流上下文时，仅显示必要的进度兼容界面。

## 6. 数据流

1. 路由参数 `[id]` 解析当前智能体。
2. 登录状态、用户角色和会话列表来自现有 Zustand stores。
3. 历史会话通过现有 conversation API 加载。
4. 输入、附件、模型、联网和绘图状态组装为现有聊天请求格式。
5. SSE 流继续通过现有流事件解析模块投影到消息状态。
6. 会话、收藏、报告和附件信息沿用当前持久化路径。
7. 视频拆解智能体继续使用现有 IndexedDB 会话视频缓存。

预览版不得引入新的聊天协议、存储前缀或智能体 ID 映射。

## 7. 异常与状态处理

- 未登录访问时沿用现有登录跳转及回跳路径。
- 对话加载时显示轻量骨架，不用整页阻塞动画。
- 发送、上传、转录和报告生成期间显示明确状态，并阻止重复提交。
- 失败时保留输入文字和未发送附件，便于重试。
- 接口超时、模型不可用和上游 HTML 错误继续使用现有中文标准化提示。
- 历史视频在当前设备不可用时，提供重新上传或仅按文字继续。
- 空历史、空收藏和无搜索结果均提供清楚的空状态。
- 不显示密钥、令牌、后端地址或其他环境变量。

## 8. 视觉规范

- 主色采用鼠尾草绿、苔藓绿和暖白，不使用蓝紫渐变。
- 页面背景与新版首页一致，但聊天卡片对比度更高，保证长文本阅读。
- 标题字体与正文建立明确层级；中文正文优先选择项目可稳定加载的字体栈。
- 圆角、阴影和动画克制，重点强调抽屉开启、消息出现和发送状态。
- 动画遵循 `prefers-reduced-motion`。
- 深色模式继续可用，但本次首先验收浅色浅绿色主题。

## 9. 响应式与无障碍

- 桌面端宽度优先验证 1440px 和 1920px。
- 移动端重点验证 390px，不允许横向滚动。
- 移动端顶部动作收进更多菜单，保留智能体名称和历史入口。
- 输入台不得遮住最新消息，并适配安全区域。
- 抽屉具备可访问名称、关闭按钮和合理焦点顺序。
- 所有图标按钮提供 `aria-label`，键盘可完成发送、打开设置和关闭抽屉。

## 10. 验收标准

### 10.1 功能

- `/chat2/35`、`/chat2/36`、`/chat2/37` 可连接真实智能体。
- 新建、继续和切换会话可用。
- 历史、收藏、删除和报告可用。
- 模型、联网和绘图设置可用并保存。
- 文件选择、拖拽、粘贴、语音和发送可用。
- 视频历史选择和缺失处理可用。
- 管理员入口仅对管理员显示。

### 10.2 视觉与交互

- 浅绿色风格与新版首页一致。
- 助手长回答以宽版文档卡片呈现。
- 历史和能力设置默认隐藏。
- 桌面端和 390px 移动端无横向溢出。
- 输入台、抽屉和消息滚动互不遮挡。

### 10.3 工程

- 当前 `/chat/[id]` 行为不变。
- ESLint 通过。
- TypeScript 检查通过。
- Next.js 生产构建通过。
- 对预览路由完成实际浏览器验收。

## 11. 正式替换路径

预览获批后执行独立变更：

1. 将新版聊天界面提升到正式 `/chat/[id]`。
2. 抽离新旧页面共用的业务控制逻辑，消除临时重复。
3. 保留旧聊天页短期回退能力。
4. 重新验证首页到聊天页的所有站内智能体入口。
5. 经确认后再提交并推送 GitHub。

本设计的完成条件是交付可独立访问、连接真实功能且不影响现有聊天页的预览版本。
